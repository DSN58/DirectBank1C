# Сценарий проверки MVP

## Подготовка

1. Создать расширение в базе 1С:Бухгалтерия 3.0.194.14.
2. Установить режим расширения `Изменение и контроль поведения`.
3. Создать объекты из `src/metadata/МетаданныеРасширения.md`.
4. Добавить общий модуль `ВП_ПравилаОбработкиВыпискиСервер` с кодом из `src/common-modules/ВП_ПравилаОбработкиВыпискиСервер.bsl`.
5. Заимствовать общий модуль `ЗагрузкаВыпискиПоБанковскомуСчету` и добавить код из `src/extension-modules/ЗагрузкаВыпискиПоБанковскомуСчету_Расширение.bsl`.

## Тестовые правила

### Правило 0: размещение депозита

- Активно: Да
- Приоритет: 5
- Направление операции: `Списание`
- Режим поиска счета: `НеЗаполнять`
- Режим заполнения: `Перезаписывать`
- Вид операции списания: `Прочее списание`
- Заполняемый контрагент: собственная организация или банк по принятому в базе способу учета
- Заполняемый банковский счет контрагента: депозитный счет
- Статья ДДС: `Размещение денежных средств во вклады`
- Счет Дт: `55.03`
- Статья прочих доходов и расходов: `Размещение денежных средств во вклады`
- Условия назначения:
  - `Содержит`: `Размещение средств в депозит`

Ожидаемый эффект: документ `СписаниеСРасчетногоСчета` создается как `Прочее списание`, с депозитным счетом и аналитикой 55.03.

- Направление операции: `Поступление`
- Режим поиска счета: `НеЗаполнять`
- Режим заполнения: `Перезаписывать`
- Вид операции поступления: `Прочее поступление`
- Заполняемый контрагент: собственная организация или банк по принятому в базе способу учета
- Заполняемый банковский счет контрагента: депозитный счет
- Статья ДДС: `Возврат депозита`
- Счет Кт: `55.03`
- Статья прочих доходов и расходов: `Возврат депозита`
- Условия назначения:
  - `Содержит`: `Возврат депозита`

Ожидаемый эффект: документ `ПоступлениеНаРасчетныйСчет` создается как `Прочее поступление`, с депозитным счетом и аналитикой 55.03.

### Правило 0.2: проценты по депозиту

- Активно: Да
- Приоритет: 7
- Направление операции: `Поступление`
- Режим поиска счета: `НеЗаполнять`
- Режим заполнения: `Перезаписывать`
- Вид операции поступления: `Прочее поступление`
- Заполняемый контрагент: банк
- Статья ДДС: `Получение процентов по депозитам`
- Счет Кт: `91.01`
- Статья прочих доходов и расходов: `Проценты к получению (уплате)` или используемая в базе статья процентов по депозитам
- Условия назначения:
  - `Содержит`: `Уплата процентов по депозиту`

Ожидаемый эффект: документ `ПоступлениеНаРасчетныйСчет` создается как `Прочее поступление`, с доходами на 91.01 и статьей процентов.

### Правило 1: оплата по номеру счета

- Активно: Да
- Приоритет: 10
- Направление операции: `Поступление`
- Режим поиска счета: `ПоНомеруВНазначении`
- Режим заполнения счета: `ТолькоЕслиПусто`
- Условия назначения:
  - `Содержит`: `ОПЛАТА`
  - `Содержит`: `СЧЕТ`
  - `НеСодержит`: `ВОЗВРАТ`
  - `НеСодержит`: `ДЕПОЗИТ`

Ожидаемый эффект: строки вроде `ОПЛАТА ПО СЧЕТУ №ЦБ-4148...` получают найденный `СчетНаОплату`.

### Правило 2: единственный неоплаченный счет покупателя

- Активно: Да
- Приоритет: 20
- Режим поиска счета: `ПоКонтрагентуЕслиОдин`
- Режим заполнения счета: `ТолькоЕслиПусто`
- Ограничитель по ИНН/контрагенту: по тестовому покупателю.
- Условия назначения:
  - `Содержит`: `ОПЛАТА`
  - `НеСодержит`: `ВОЗВРАТ`

Ожидаемый эффект: счет подставляется только если у распознанного покупателя найден ровно один неоплаченный счет.

## Проверка на файле `kl_to_1c.txt`

Файл используется как удобный источник строк назначения платежа и ручной тест чтения формата 1С.

Важно: правила теперь применяются и к файловой загрузке, и к DirectBank. Ограничение DirectBank снято; проверку удобно начинать с файла `kl_to_1c.txt`.

Проверяемые назначения из файла:

- `Размещение средств в депозит по Ген.согл №ДЕППИБ-23-724910-141...`
- `Возврат депозита по Ген.согл №ДЕППИБ-23-724910-141...`
- `Уплата процентов по депозиту с 24.04.2026...`
- `ОПЛАТА ПО СЧЕТУ №ЦБ-4148 ОТ 23.04.2026Г. ЗА МАТРАСЫ...`
- `ОПЛАТА ЗА МАТРАСЫ ПО СЧЕТУ №Б/Н ОТ 27.04.2026Г...`

## Проверка DirectBank

1. Включить расширение.
2. Создать или подобрать покупателя, договор и счет на оплату покупателю.
3. Настроить правило.
4. Загрузить выписку через DirectBank так, чтобы документ `ПоступлениеНаРасчетныйСчет` создавался автоматически.
5. Проверить:
   - создан новый документ `ПоступлениеНаРасчетныйСчет`;
   - вид операции: `Оплата покупателя`;
   - в `РасшифровкаПлатежа.СчетНаОплату` заполнен ожидаемый счет;
   - в `ВП_ЖурналПримененияПравилВыписки` есть запись с результатом `Применено`.

## Негативные проверки

- Если документ уже найден типовой загрузкой, правило не должно менять его.
- Если есть платежное поручение, правило не должно применяться.
- Если режим `ТолькоЕслиПусто`, а типовая БП уже нашла счет, счет не перезаписывается.
- Если по контрагенту найдено несколько неоплаченных счетов, режим `ПоКонтрагентуЕслиОдин` не выбирает счет.
- Если включен будущий режим `РаспределитьПоНесколькимСчетам`, журнал фиксирует отказ применения.
