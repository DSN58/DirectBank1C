# Метаданные расширения

Префикс объектов расширения: `ВП_`.

## Общие свойства расширения

- Назначение: автоматическое заполнение реквизитов банковских документов при создании новых документов из выписки банка и DirectBank.
- Режим совместимости: по базе БП 3.0.194.14.
- Режим расширения: `Изменение и контроль поведения`.

## Перечисление `ВП_РежимыПоискаСчетаНаОплату`

Значения:

- `НеЗаполнять`
- `КонкретныйСчет`
- `ПоНомеруВНазначении`
- `ПоКонтрагентуЕслиОдин`
- `РаспределитьПоНесколькимСчетам`

`РаспределитьПоНесколькимСчетам` добавляется как задел под будущую версию; в MVP логика для него возвращает отказ применения.
`НеЗаполнять` используется для правил, которые заполняют другие реквизиты банковского документа и не работают со счетом на оплату.

## Перечисление `ВП_НаправленияОперацийВыписки`

Значения:

- `Любая`
- `Поступление`
- `Списание`

## Перечисление `ВП_РежимыЗаполненияСчетаНаОплату`

Значения:

- `ТолькоЕслиПусто`
- `Перезаписывать`

## Перечисление `ВП_ВидыУсловийНазначенияПлатежа`

Значения:

## Справочник `ВП_ПравилаОбработкиБанковскойВыписки`

Назначение: пользовательские правила заполнения реквизитов банковского документа при загрузке выписки.

Реквизиты:

| Имя | Тип | Обязательный | Описание |
| --- | --- | --- | --- |
| `Активно` | Булево | Да | Правило участвует в подборе |
| `Приоритет` | Число 10,0 | Да | Меньшее значение применяется раньше |
| `НаправлениеОперации` | `ПеречислениеСсылка.ВП_НаправленияОперацийВыписки` | Да | Любая операция, поступление или списание |
| `Организация` | `СправочникСсылка.Организации` | Нет | Ограничитель по организации |
| `БанковскийСчетОрганизации` | `СправочникСсылка.БанковскиеСчета` | Нет | Ограничитель по банковскому счету |
| `ВидОперацииДокумента` | `ПеречислениеСсылка.ВидыОперацийПоступлениеДенежныхСредств` | Нет | Ограничитель по виду операции |
| `Контрагент` | `СправочникСсылка.Контрагенты` | Нет | Ограничитель по распознанному покупателю |
| `ИННПлательщика` | Строка 12 | Нет | Ограничитель по ИНН из банковской операции |
| `РежимПоискаСчета` | `ПеречислениеСсылка.ВП_РежимыПоискаСчетаНаОплату` | Да | Как определить счет |

| `СчетНаОплатуПокупателю` | `ДокументСсылка.СчетНаОплатуПокупателю` | Нет | Используется при режиме `КонкретныйСчет` |
| `ВидОперацииПоступление` | `ПеречислениеСсылка.ВидыОперацийПоступлениеДенежныхСредств` | Нет | Вид операции для документа `ПоступлениеНаРасчетныйСчет` |
| `ВидОперацииСписание` | `ПеречислениеСсылка.ВидыОперацийСписаниеДенежныхСредств` | Нет | Вид операции для документа `СписаниеСРасчетногоСчета` |
| `ЗаполняемыйКонтрагент` | `СправочникСсылка.Контрагенты` | Нет | Контрагент/плательщик/получатель, который нужно подставить |
| `ЗаполняемыйДоговорКонтрагента` | `СправочникСсылка.ДоговорыКонтрагентов` | Нет | Договор, который нужно подставить |
| `ЗаполняемыйБанковскийСчетКонтрагента` | `СправочникСсылка.БанковскиеСчета` | Нет | Банковский счет контрагента или депозитный счет |
| `СтатьяДвиженияДенежныхСредств` | `СправочникСсылка.СтатьиДвиженияДенежныхСредств` | Нет | Статья ДДС |
| `СчетДт` | `ПланСчетовСсылка.Хозрасчетный` | Нет | Счет дебета для прочих списаний и нестандартных операций |
| `СчетКт` | `ПланСчетовСсылка.Хозрасчетный` | Нет | Счет кредита для прочих поступлений и нестандартных операций |
| `ПодразделениеДт` | `СправочникСсылка.ПодразделенияОрганизаций` | Нет | Подразделение дебета |
| `ПодразделениеКт` | `СправочникСсылка.ПодразделенияОрганизаций` | Нет | Подразделение кредита |
| `СтатьяПрочихДоходовИРасходов` | `СправочникСсылка.ПрочиеДоходыИРасходы` | Нет | Субконто для счетов 91.01/91.02 |

Табличная часть `УсловияНазначенияПлатежа`:

| Имя | Тип | Описание |
| --- | --- | --- |
| `ВидУсловия` | `ПеречислениеСсылка.ВП_ВидыУсловийНазначенияПлатежа` | `Содержит` или `НеСодержит` |
| `СтрокаПоиска` | Строка 250 | Фрагмент назначения платежа |
| `БезУчетаРегистра` | Булево | Сравнивать без учета регистра |

Рекомендуемые настройки формы элемента:

- В шапке: `Активно`, `Приоритет`, `Наименование`.
- Группа `Ограничители`: организация, банковский счет, вид операции, контрагент, ИНН.
- Группа `Действие`: направление, режим заполнения, вид операции поступления/списания, контрагент, договор, банковский счет, статьи, счета Дт/Кт, конкретный счет.
- Таблица `УсловияНазначенияПлатежа`.
- В модуль формы элемента вставить код из `src/catalog-forms/ВП_ПравилаОбработкиБанковскойВыписки.ФормаЭлемента.Модуль.txt`.

Обработчики формы элемента:

- `ПриОткрытии`
- `РежимПоискаСчетаПриИзменении`
- `НаправлениеОперацииПриИзменении`
- `РежимЗаполненияСчетаПриИзменении`

Модуль формы управляет доступностью:

- `СчетНаОплатуПокупателю` доступен только при режиме `КонкретныйСчет`;
- `ВидОперацииПоступление`, `СчетКт`, `ПодразделениеКт` доступны для `Поступление` и `Любая`;
- `ВидОперацииСписание`, `СчетДт`, `ПодразделениеДт` доступны для `Списание` и `Любая`.

## Расширение общего модуля `ЗагрузкаВыпискиПоБанковскомуСчету`

В расширении заимствовать общий модуль `ЗагрузкаВыпискиПоБанковскомуСчету` и добавить обработчик `&Вместо("СоздатьДокументы")`.

Назначение обработчика:

1. Перед вызовом типовой `СоздатьДокументы()` применить правила к новым операциям выписки.
2. Передать управление типовой логике через `ПродолжитьВызов()`.
3. После штатного создания повторно применить действия правил к созданным документам, чтобы заполнить реквизиты, которые типовая логика не перенесла из `ХозяйственнаяОперация`.
